# BarberSync 
**Purpose:** Digitalize appointment booking for a 2-man shop with zero-friction (no accounts).

## Tech Stack (Planned)
- **Backend:** Django (Python)
- **Database:** PostgreSQL (via Supabase)
- **Frontend:** Responsive Web (Mobile-First)

## Core Logic Requirements
- 3-week rolling booking window (updates at midnight).
- "Soft Hold" system (10-minute expiry for unconfirmed slots).
- Unique Ref ID generation (e.g., APT-12345) to ensure Zero PII.
